# Requirements Document

## 目的

Discordサーバー上で、ユーザー同士が複数種類の「ポイント」を付与できるBotを提供する（SPEC.md §1参照）。

## ユースケース

以下の3つのコマンドを提供する（SPEC.md §5参照）：

- `/give` - 他ユーザーにポイントを付与
- `/points` - 自分の現在のポイントを確認  
- `/leaderboard` - ポイントランキングを表示

## 制約条件

- **付与量**: 1回あたり 1〜100ポイント（SPEC.md §4.1参照）
- **日次送信回数制限**: 送信者×ポイント種×日付単位で制限（SPEC.md §5.1参照）
- **禁止事項**: 自分自身・Botアカウントへの付与は禁止（SPEC.md §5.1参照）
- **日付基準**: JST（UTC+9）基準（SPEC.md §7参照）

## 受け入れ基準

- /give により受信者の残高が増える（SPEC.md §8参照）
- バリデーションエラーで適切に失敗する（SPEC.md §8参照）
- 日次送信回数上限を超えると失敗し、残高は増えない（SPEC.md §8参照）
- /points で残高確認できる（SPEC.md §8参照）
- /leaderboard で上位10が表示される（SPEC.md §8参照）
- Bot再起動後もデータが保持される（SPEC.md §8参照）
